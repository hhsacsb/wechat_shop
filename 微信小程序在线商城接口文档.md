# 微信小程序在线商城接口文档

## 1. 文档说明

### 1.1 文档目的
本文档用于定义在线商城微信小程序一期项目的后端接口规范，供前端开发、后端开发、测试联调及后续维护使用。

### 1.2 适用范围
- 微信小程序客户端接口
- 商城前台业务接口
- 一期核心交易流程相关接口

### 1.3 接口风格
- 接口协议：`HTTPS`
- 数据格式：`JSON`
- 字符编码：`UTF-8`
- 接口风格：`RESTful`

## 2. 基础规范

### 2.1 基础域名
以下域名为示例，实际以部署环境为准：

```text
开发环境：https://dev-api.xxx.com
测试环境：https://test-api.xxx.com
生产环境：https://api.xxx.com
```

### 2.2 公共请求头

| 请求头 | 是否必填 | 说明 |
| --- | --- | --- |
| Content-Type | 是 | `application/json` |
| Authorization | 否 | 用户登录后传 `Bearer token` |
| X-Client-Type | 否 | 客户端类型，如 `miniapp` |
| X-App-Version | 否 | 小程序版本号 |

### 2.3 通用返回结构

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

### 2.4 返回字段说明

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| code | int | 业务状态码，`0` 表示成功 |
| message | string | 返回消息 |
| data | object/array | 返回业务数据 |

### 2.5 分页返回结构

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "list": [],
    "page": 1,
    "page_size": 10,
    "total": 100,
    "has_more": true
  }
}
```

## 3. 鉴权说明

### 3.1 登录方式
- 小程序前端通过 `wx.login` 获取 `code`
- 后端通过微信接口换取 `openid`、`session_key`
- 后端生成平台登录态 `token`
- 前端后续请求通过 `Authorization: Bearer token` 传递身份信息

### 3.2 无需登录接口
- 首页商品列表
- 商品分类
- 商品列表
- 商品详情
- 商品搜索

### 3.3 需要登录接口
- 用户信息
- 地址管理
- 购物车
- 订单
- 售后

## 4. 业务状态码

| 状态码 | 说明 |
| --- | --- |
| 0 | 成功 |
| 40001 | 请求参数错误 |
| 40002 | 缺少必要参数 |
| 40003 | 参数格式错误 |
| 40101 | 未登录或 token 失效 |
| 40102 | 无访问权限 |
| 40401 | 数据不存在 |
| 40901 | 商品库存不足 |
| 40902 | 订单状态不允许当前操作 |
| 40903 | 重复提交请求 |
| 50001 | 系统内部错误 |
| 50002 | 第三方服务调用失败 |
| 50003 | 支付处理失败 |

## 5. 用户模块

### 5.1 微信登录

#### 接口信息
- 接口地址：`POST /api/user/wx-login`
- 是否鉴权：否
- 接口说明：使用微信登录凭证换取平台登录态

#### 请求参数

| 字段 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| code | string | 是 | 微信 `wx.login` 返回的 code |
| nickname | string | 否 | 用户昵称 |
| avatar | string | 否 | 用户头像 |

#### 请求示例

```json
{
  "code": "021A1abcXYZ",
  "nickname": "微信用户",
  "avatar": "https://cdn.xxx.com/avatar/default.png"
}
```

#### 返回示例

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiJ9.demo.token",
    "user_id": 10001,
    "openid": "oAbcdEFG123456",

}
```

## 12. 联调建议

- 前后端统一约定金额字段使用 `decimal(10,2)` 语义，接口中返回数值类型
- 所有需要鉴权的接口统一从请求头读取 `Authorization`
- 订单创建、支付回调、库存扣减必须保证幂等性
- 商品列表、订单列表、售后列表统一采用分页结构，便于前端组件复用
- 接口上线前建议通过 Swagger 或 Apifox 再生成一版可调试文档
